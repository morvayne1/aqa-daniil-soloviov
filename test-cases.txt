Test Cases for Restful-Booker Platform (automationintesting.online)
Preconditions for all tests: Application is opened on the home page

Successful booking with valid data
Steps:
1. Navigate to Rooms
2. Select "Single" room and click "Book Now"
3. Select valid future dates (e.g., 20–25 of current month)
4. Click "Reserve Now"
5. Fill guest details (Firstname, Lastname, Email, Phone)
6. Submit booking
Expected Result: "Booking Confirmed" message appears

Attempt to book on invalid (past/overlapping) dates
Steps:
1. Navigate to Rooms
2. Select "Single" room and click "Book Now"
3. Select past or overlapping dates
4. Fill guest details and submit
5. Expected Result: No confirmation message, form remains open
Actual Result: Frontend may crash ("Application error") or booking succeeds — BUG

Check visibility of booked dates
Steps:
1. Open booking calendar for any room
2. Navigate to month with existing bookings
3. Expected Result: Booked dates have "Unavailable" title
Actual Result: Title visible — PASS

Check if booked dates are selectable
Steps:
1. Open booking calendar for any room
2. Try to select dates marked as unavailable
Expected Result: Dates should be non-interactive
Actual Result: Dates are selectable — BUG
